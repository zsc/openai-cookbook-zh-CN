示例：微调分类器

我们将微调一个 babbage-002 分类器（ada 模型的替代品），以区分棒球和曲棍球这两种运动。

from sklearn.datasets import fetch_20newsgroups
import pandas as pd
import openai
import os

client = openai.OpenAI(api_key=os.environ.get("OPENAI_API_KEY", "<如果未设置环境变量，则为您的 OpenAI API 密钥>"))

categories = ['rec.sport.baseball', 'rec.sport.hockey']
sports_dataset = fetch_20newsgroups(subset='train', shuffle=True, random_state=42, categories=categories)

数据准备

我们将数据集转换为 pandas 数据框，其中包含 prompt 和 completion 列。prompt 包含来自邮件列表的电子邮件，completion 是运动的名称，即曲棍球或棒球。出于演示目的和微调速度的考虑，我们只取 300 个样本。在实际用例中，样本越多，性能越好。

import pandas as pd

labels = [sports_dataset.target_names[x].split('.')[-1] for x in sports_dataset['target']]
texts = [text.strip() for text in sports_dataset['data']]
df = pd.DataFrame(zip(texts, labels), columns = ['prompt','completion']) #[:300]

该工具会很有帮助地建议对数据集进行一些改进，并将数据集拆分为训练集和验证集。

在 prompt 和 completion 之间添加后缀分隔符对于告知模型输入文本已停止，并且现在需要预测类别至关重要。由于我们在每个示例中使用相同分隔符，模型可以学习到它应该在分隔符之后预测棒球或曲棍球。 在 completion 前添加空格前缀很有用，因为大多数词元都以空格前缀进行标记。 该工具还识别出这很可能是一个分类任务，因此建议将数据集拆分为训练集和验证集。这将使我们能够轻松地在新数据上衡量预期性能。

微调

该工具建议我们运行以下命令来训练数据集。由于这是一个分类任务，我们希望了解我们的分类用例在提供的验证集上的泛化性能。

我们可以直接从 CLI 工具复制建议的命令。我们特别添加 -m ada 来微调一个更便宜、更快的 ada 模型，该模型在分类用例上的性能通常与更慢、更昂贵的模型相当。

train_file = client.files.create(file=open("sport2_prepared_train.jsonl", "rb"), purpose="fine-tune")

我们需要在 prompt 后面使用与微调期间相同的分隔符。在这种情况下，它是 \n\n###\n\n。由于我们关心的是分类，我们希望温度尽可能低，并且只需要一个词元的 completion 来确定模型的预测。

ft_model = fine_tune_results.fine_tuned_model

# 注意，这会调用旧的 completions API - https://platform.openai.com/docs/api-reference/completions
res = client.completions.create(model=ft_model, prompt=test['prompt'][0] + '\n\n###\n\n', max_tokens=1, temperature=0)
res.choices[0].text

' hockey'

要获取对数概率，我们可以在 completion 请求中指定 logprobs 参数。

res = client.completions.create(model=ft_model, prompt=test['prompt'][0] + '\n\n###\n\n', max_tokens=1, temperature=0, logprobs=2)
res.choices[0].logprobs.top_logprobs

[{' hockey': 0.0, ' Hockey': -22.504879}]

我们可以看到，模型预测曲棍球的可能性远大于棒球，这是正确的预测。通过请求 log_probs，我们可以看到每个类别的预测（对数）概率。
