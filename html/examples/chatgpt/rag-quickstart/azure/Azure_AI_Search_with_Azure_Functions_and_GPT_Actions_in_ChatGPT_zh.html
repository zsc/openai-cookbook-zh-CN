全文逐字翻译成中文（只输出翻译后结果）： # Azure AI Search 作为向量数据库 + Azure Functions 用于与 ChatGPT 集成 GPT

本笔记本提供了使用 Azure AI Search（以前称为 Azure Cognitive Search）作为向量数据库和 OpenAI 嵌入的详细分步说明，然后在此基础上创建一个 Azure Function，以便在 ChatGPT 中插入自定义 GPT。

这可以为希望在 Azure 中设置 RAG 基础设施的客户提供解决方案，并将其公开为与 ChatGPT 等其他平台集成的端点。

Azure AI Search 是一项云搜索服务，为开发人员提供了用于在 Web、移动和企业应用程序中基于私有、异构内容构建丰富搜索体验的基础设施、API 和工具。

Azure Functions 是一项无服务器计算服务，可运行事件驱动的代码，自动管理基础设施、扩展并与其他 Azure 服务集成。

先决条件：

为了完成此练习，您必须具备以下条件：

  • 具有创建 Azure AI Search 服务 和 Azure Function 应用权限的 Azure 用户
  • Azure 订阅 ID 和资源组。
  • OpenAI 密钥

架构

下面是此解决方案架构的图示，我们将逐步进行介绍。

注意：这种向量数据存储 + 无服务器函数架构模式可以推广到其他向量数据存储。例如，如果您想在 Azure 中使用类似 Postgres 的产品，您可以将 配置 Azure AI Search 设置 步骤更改为设置 Postgres 的要求，将 创建 Azure AI Vector Search 更改为在 Postgres 中创建数据库和表，并更新此存储库中的 function_app.py 代码以查询 Postgres 而不是 Azure AI Search。数据准备和 Azure Function 的创建将保持一致。

目录：

  1. 设置环境 通过安装和导入所需的库以及配置我们的 Azure 设置来设置环境。包括：

    • 安装和导入所需库
    • 配置 OpenAI 设置
    • 配置 Azure AI Search 设置

  2. 准备数据 通过嵌入文档以及捕获其他元数据来准备要上传的数据。我们将使用 OpenAI 文档的一个子集作为示例数据。

  3. 创建 Azure AI Vector Search 创建一个 Azure AI Vector Search 并上传我们准备好的数据。包括：

    • 创建索引：在 Azure AI Search 中创建索引的步骤。
    • 上传数据：将数据上传到 Azure AI Search 的说明。
    • 测试搜索：测试搜索功能的步骤。

  4. 创建 Azure Function 创建一个 Azure Function 来与 Azure AI Vector Search 进行交互。包括：

    • 创建存储帐户：创建 Azure Function 的存储帐户的步骤。
    • 创建 Function App：在 Azure 中创建 Function App 的说明。

  5. 在 ChatGPT 的自定义 GPT 中输入 将 Azure Function 与 ChatGPT 中的自定义 GPT 集成。包括：

    • 创建 OpenAPI 规范：创建 Azure Function 的 OpenAPI 规范的步骤。
    • 创建 GPT 指令：创建用于集成的 GPT 特定指令的说明。

# 使用指定的参数创建或更新搜索服务

为了处理超过 8191 个令牌的上下文长度的长文本文件，我们可以分别使用分块嵌入，或者以某种方式组合它们，例如平均（按每个块的大小加权）。

我们将从 Python 自己的 cookbook 中获取一个函数，该函数将序列分解成块。

def batched(iterable, n):
    """将数据分批成长度为 n 的元组。最后一个批次可能较短。"""
    # batched('ABCDEFG', 3) --> ABC DEF G
    if n < 1:
        raise ValueError('n 必须至少为一')
    it = iter(iterable)
    while (batch := tuple(islice(it, n))):
        yield batch

现在我们定义一个函数，该函数将字符串编码为令牌，然后将其分解成块。我们将使用 tiktoken，这是 OpenAI 的一个快速开源分词器。

要了解有关使用 Tiktoken 计算令牌的更多信息，请参阅 此 cookbook。

def chunked_tokens(text, chunk_length, encoding_name='cl100k_base'):
    # 获取指定编码名称的编码对象。OpenAI 的 tiktoken 库（在本笔记本中使用）目前支持两种编码：“bpe”和“cl100k_base”。“bpe”编码用于 GPT-3 及更早的模型，“cl100k_base”用于 GPT-4 等较新模型。
    encoding = tiktoken.get_encoding(encoding_name)
    # 将输入文本编码为令牌
    tokens = encoding.encode(text)
    # 创建一个迭代器，该迭代器产生指定长度的令牌块
    chunks_iterator = batched(tokens, chunk_length)
    # 从迭代器中产生每个块
    yield from chunks_iterator

最后，我们可以编写一个函数来安全地处理嵌入请求，即使输入文本的长度超过最大上下文长度，也可以通过单独嵌入每个块来处理。可以将 average 标志设置为 True 以返回分块嵌入的加权平均值，或设置为 False 以仅返回未修改的分块嵌入列表。

注意：您可以在此处采用其他更复杂的技术，包括：

  • 使用 GPT-4o 捕获图像/图表描述以进行嵌入。
  • 在块之间保持文本重叠以最大程度地减少重要上下文的截断。
  • 按段落或部分进行分块。
  • 添加有关每个文章的更具描述性的元数据。
## 根据模型更改以下内容。以下是 OpenAI 的最新嵌入模型，因此您可以将其保留原样，除非您使用的是不同的嵌入模型。
EMBEDDING_CTX_LENGTH = 8191
EMBEDDING_ENCODING='cl100k_base'

def generate_embeddings(text, model):
    # 使用指定的模型为提供的文本生成嵌入
    embeddings_response = openai_client.embeddings.create(model=model, input=text)
    # 从响应中提取嵌入数据
    embedding = embeddings_response.data[0].embedding
    return embedding

def len_safe_get_embedding(text, model=embeddings_model, max_tokens=EMBEDDING_CTX_LENGTH, encoding_name=EMBEDDING_ENCODING):
    # 初始化用于存储嵌入和相应文本块的列表
    chunk_embeddings = []
    chunk_texts = []
    # 迭代输入文本的令牌块
    for chunk in chunked_tokens(text, chunk_length=max_tokens, encoding_name=encoding_name):
        # 为每个块生成嵌入并附加到列表中
        chunk_embeddings.append(generate_embeddings(chunk, model=model))
        # 将块解码回文本并附加到列表中
        chunk_texts.append(tiktoken.get_encoding(encoding_name).decode(chunk))
    # 返回块嵌入列表和相应的文本块
    return chunk_embeddings, chunk_texts

现在，我们可以定义一些辅助函数来处理 data 文件夹中的 oai_docs 文件夹中的 .txt 文件。您也可以将其与自己的数据一起使用，它支持 .txt 和 .pdf 文件。

def extract_text_from_pdf(pdf_path):
    # 初始化 PDF 阅读器
    reader = PdfReader(pdf_path)
    text = ""
    # 遍历 PDF 中的每一页并提取文本
    for page in reader.pages:
        text += page.extract_text()
    return text

def process_file(file_path, idx, categories, embeddings_model):
    file_name = os.path.basename(file_path)
    print(f"正在处理文件 {idx + 1}: {file_name}")

    # 从 .txt 文件读取文本内容
    if file_name.endswith('.txt'):
        with open(file_path, 'r', encoding='utf-8') as file:
            text = file.read()
    # 从 .pdf 文件提取文本内容
    elif file_name.endswith('.pdf'):
        text = extract_text_from_pdf(file_path)

    title = file_name
    # 为标题生成嵌入
    title_vectors, title_text = len_safe_get_embedding(title, embeddings_model)
    print(f"为 {file_name} 生成了标题嵌入")

    # 为内容生成嵌入
    content_vectors, content_text = len_safe_get_embedding(text, embeddings_model)
    print(f"为 {file_name} 生成了内容嵌入")

    category = categorize_text(' '.join(content_text), categories)
    print(f"将 {file_name} 分类为 {category}")

    # 准备要附加的数据
    data = []
    for i, content_vector in enumerate(content_vectors):
        data.append({
            "id": f"{idx}_{i}",
            "vector_id": f"{idx}_{i}",
            "title": title_text[0],
            "text": content_text[i],
            "title_vector": json.dumps(title_vectors[0]),  # 假设标题很短，只有一个块
            "content_vector": json.dumps(content_vector),
            "category": category
        })
        print(f"已附加 {file_name} 的第 {i + 1}/{len(content_vectors)} 个块的数据")

    return data
