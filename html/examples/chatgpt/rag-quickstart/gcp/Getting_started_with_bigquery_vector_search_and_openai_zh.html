GCP BigQuery 与 GCP Functions 和 ChatGPT 中的 GPT 操作

本笔记本提供了使用 Google Cloud BigQuery 作为具有向量搜索功能的数据库的分步说明，结合 OpenAI 嵌入，然后在其之上创建一个 Google Cloud Function 以集成到 ChatGPT 的自定义 GPT 中。

这可以为希望在 Google Cloud Platform (GCP) 内设置 RAG 基础设施的客户提供解决方案，并将其作为端点暴露，以便与其他平台（如 ChatGPT）集成。

Google Cloud BigQuery 是一个完全托管的无服务器数据仓库，可利用 Google 基础架构的处理能力实现超快速的 SQL 查询。它使开发人员能够轻松地存储和分析海量数据集。

Google Cloud Functions 是一个轻量级的、事件驱动的异步计算解决方案，允许您创建响应云事件的小型、单一用途函数，而无需管理服务器或运行时环境。

先决条件：

要运行此食谱，您必须拥有：

  • 您有权访问的 GCP 项目
  • 具有创建 BigQuery 数据集和 Google Cloud Function 权限的 GCP 用户
  • 已安装并连接的 GCP CLI
  • OpenAI API 密钥
  • ChatGPT Plus、Teams 或 Enterprise 订阅

架构

以下是此解决方案架构的图表，我们将逐步进行介绍：

目录

  1. 设置环境 通过安装和导入所需的库以及配置我们的 GCP 设置来设置环境。包括：

    • 安装和导入所需库
    • 配置 GCP 项目
    • 配置 OpenAI 设置

  2. 准备数据 通过嵌入文档以及捕获其他元数据来准备要上传的数据。我们将使用 OpenAI 文档的一个子集作为示例数据。

  3. 创建具有向量搜索的 BigQuery 表 创建 BigQuery 表并上传我们准备好的数据。包括：

    • 创建数据集：创建 BigQuery 数据集的步骤。
    • 创建表并上传数据：创建 BigQuery 表的说明。

  4. 创建 GCP 函数 使用 gcloud CLI 和先前计算的环境变量

  5. 在 ChatGPT 的自定义 GPT 中输入 在 BigQuery 中嵌入的数据上执行搜索：

    • 向量搜索：执行基于向量的搜索查询的步骤。
    • 元数据过滤搜索：进行元数据过滤的说明。

准备数据

我们将嵌入并存储 oai_docs 文件夹中的 OpenAI 文档的几页。我们将首先嵌入每个文档，将其添加到 CSV 中，然后使用该 CSV 上传到索引。

我们将使用 此食谱 中突出显示的一些技术。这是一种快速嵌入文本的方法，不考虑分段、使用我们的视觉模型描述图像/图形/图表、长文档之间的文本重叠等变量。

为了处理超过 8191 个令牌上下文的更长文本文件，我们可以单独使用分块嵌入，或者以某种方式组合它们，例如平均（按每个块的大小加权）。

我们将从 Python 自己的食谱中采用一个函数，该函数将序列分解成块。

def batched(iterable, n):
    """将数据分批成长度为 n 的元组。最后一个批次可能较短。"""
    # batched('ABCDEFG', 3) --> ABC DEF G
    if n < 1:
        raise ValueError('n 必须至少为一')
    it = iter(iterable)
    while (batch := tuple(islice(it, n))):
        yield batch

现在我们定义一个函数，该函数将字符串编码为令牌，然后将其分解成块。我们将使用 tiktoken，这是 OpenAI 的快速开源分词器。

要了解有关使用 Tiktoken 计算令牌的更多信息，请参阅 此食谱。

def chunked_tokens(text, chunk_length, encoding_name='cl100k_base'):
    # 获取指定编码名称的编码对象。OpenAI 的 tiktoken 库（在本笔记本中使用）目前支持两种编码：“bpe”和“cl100k_base”。“bpe”编码用于 GPT-3 及更早的模型，“cl100k_base”用于 GPT-4 等较新模型。
    encoding = tiktoken.get_encoding(encoding_name)
    # 将输入文本编码为令牌
    tokens = encoding.encode(text)
    # 创建一个迭代器，该迭代器产生指定长度的令牌块
    chunks_iterator = batched(tokens, chunk_length)
    # 从迭代器中产生每个块
    yield from chunks_iterator

最后，我们可以编写一个函数来安全地处理嵌入请求，即使输入文本长于最大上下文长度，方法是分块输入令牌并单独嵌入每个块。平均标志可以设置为 True 以返回块嵌入的加权平均值，或设置为 False 以简单地返回未修改的块嵌入列表。

注意：您可以在此处采用其他技术，包括：

  • 使用 GPT-4o 捕获图像/图表描述以进行嵌入
  • 按段落或部分进行分块
  • 添加有关每篇文章的更多描述性元数据。
EMBEDDING_CTX_LENGTH = 8191
EMBEDDING_ENCODING='cl100k_base'

def generate_embeddings(text, model):
    # 使用指定的模型为提供的文本生成嵌入
    embeddings_response = openai_client.embeddings.create(model=model, input=text)
    # 从响应中提取嵌入数据
    embedding = embeddings_response.data[0].embedding
    return embedding

def len_safe_get_embedding(text, model=embeddings_model, max_tokens=EMBEDDING_CTX_LENGTH, encoding_name=EMBEDDING_ENCODING):
    # 初始化用于存储嵌入和相应文本块的列表
    chunk_embeddings = []
    chunk_texts = []
    # 迭代输入文本的令牌块
    for chunk in chunked_tokens(text, chunk_length=max_tokens, encoding_name=encoding_name):
        # 为每个块生成嵌入并附加到列表中
        chunk_embeddings.append(generate_embeddings(chunk, model=model))
        # 将块解码回文本并附加到列表中
        chunk_texts.append(tiktoken.get_encoding(encoding_name).decode(chunk))
    # 返回块嵌入列表和相应的文本块
    return chunk_embeddings, chunk_texts

categories = ['authentication','models','techniques','tools','setup','billing_limits','other']

def categorize_text(text, categories):

    # 创建用于分类的提示
    messages = [
        {"role": "system", "content": f"""您是 LLM 专家，您将获得对应于 OpenAI 文档中一篇文章的文本。
         将文档分类到以下类别之一：{', '.join(categories)}。只回复类别名称，不要回复其他任何内容。"""},
        {"role": "user", "content": text}
    ]
    try:
        # 调用 OpenAI API 对文本进行分类
        response = openai_client.chat.completions.create(
            model="gpt-4o",
            messages=messages
        )

        # 从响应中提取类别
        category = response.choices[0].message.content
        return category
    except Exception as e:
        print(f"分类文本时出错：{str(e)}")
        return None

现在，我们可以定义一些辅助函数来处理 oai_docs 文件夹中的 .txt 文件。请随时将其用于您自己的数据，它同时支持 .txt 和 .pdf 文件。

def extract_text_from_pdf(pdf_path):
    # 初始化 PDF 阅读器
    reader = PdfReader(pdf_path)
    text = ""
    # 遍历 PDF 中的每一页并提取文本
    for page in reader.pages:
        text += page.extract_text()
    return text

def process_file(file_path, idx, categories, embeddings_model):
    file_name = os.path.basename(file_path)
    print(f"正在处理文件 {idx + 1}: {file_name}")

    # 从 .txt 文件读取文本内容
    if file_name.endswith('.txt'):
        with open(file_path, 'r', encoding='utf-8') as file:
            text = file.read()
    # 从 .pdf 文件提取文本内容
    elif file_name.endswith('.pdf'):
        text = extract_text_from_pdf(file_path)

    title = file_name
    # 为标题生成嵌入
    title_vectors, title_text = len_safe_get_embedding(title, embeddings_model)
    print(f"已为 {file_name} 生成标题嵌入")

    # 为内容生成嵌入
    content_vectors, content_text = len_safe_get_embedding(text, embeddings_model)
    print(f"已为 {file_name} 生成内容嵌入")

    category = categorize_text(' '.join(content_text), categories)
    print(f"已将 {file_name} 分类为 {category}")

    # 准备要附加的数据
    data = []
    for i, content_vector in enumerate(content_vectors):
        data.append({
            "id": f"{idx}_{i}",
            "vector_id": f"{idx}_{i}",
            "title": title_text[0],
            "text": content_text[i],
            "title_vector": json.dumps(title_vectors[0]),  # 假设标题很短，只有一个块
            "content_vector": json.dumps(content_vector),
            "category": category
        })
        print(f"已附加 {file_name} 的 {i + 1}/{len(content_vectors)} 个块的数据")

    return data
