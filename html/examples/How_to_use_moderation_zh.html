如何使用审核 API

注意： 本指南旨在补充我们的“护栏食谱”，通过更专注于审核技术来提供更深入的见解。虽然内容和结构上存在一些重叠，但本食谱更深入地探讨了根据特定需求定制审核标准的细微差别，提供了更精细级别的控制。如果您对内容安全措施（包括护栏和审核）有更广泛的概述，我们建议从“护栏食谱”开始。这些资源共同提供了对如何在应用程序中有效管理和审核内容的全面理解。

审核就像现实世界中的护栏一样，是一种预防措施，可确保您的应用程序保持在可接受和安全内容的范围内。审核技术非常通用，可以应用于 LLM 可能遇到问题的各种场景。本笔记本旨在提供可以直接改编以满足您特定需求的简单示例，同时还讨论了决定是否实施审核以及如何进行的注意事项和权衡。本笔记本将使用我们的 审核 API，这是一个可用于检查文本或图像是否可能有害的工具。

本笔记本将重点关注：

  • 输入审核： 在 LLM 处理不当或有害内容之前对其进行识别和标记。
  • 输出审核： 在 LLM 生成的内容到达最终用户之前对其进行审查和验证。
  • 自定义审核： 定制审核标准和规则以满足您应用程序的特定需求和上下文，确保个性化和有效的内​​容控制机制。
from openai import OpenAI
client = OpenAI()
GPT_MODEL = 'gpt-4o-mini'

1. 输入审核

输入审核侧重于防止有害或不当内容到达 LLM，常见应用包括：

  • 内容过滤： 防止在社交媒体、论坛和内容创建平台上散播仇恨言论、骚扰、露骨材料和错误信息。
  • 社区标准执行： 确保用户互动（例如评论、论坛帖子和聊天消息）遵守在线平台的社区准则和标准，包括教育环境、游戏社区或约会应用程序。
  • 垃圾邮件和欺诈预防： 过滤在线论坛、评论区、电子商务平台和客户评论中的垃圾邮件、欺诈内容和误导性信息。

这些措施充当预防性控制，在 LLM 之前或与 LLM 一起运行，以在满足特定标准时更改应用程序的行为。

拥抱异步

一种常见的最小化延迟的设计是异步发送审核与您的主要 LLM 调用一起发送。如果触发了审核，则发送占位符响应，否则发送 LLM 响应。此模式也可以在我们的 护栏食谱 中找到。值得注意的是，虽然异步模式在最小化延迟方面很有效，但它也可能导致不必要的成本。具体来说，如果内容在处理之前被标记，您可以避免完成成本。因此，在使用异步模式时，平衡减少延迟的好处与潜在的费用增加至关重要。

我们将使用此方法，创建一个 execute_chat_with_moderation 函数，该函数将并行运行 LLM 的 get_chat_response 和 check_expression 审核函数，并且仅当审核返回 False（未触发）时才返回 LLM 响应。

工作流程：

我们将创建一个工作流程，其中包含审核 API，用于在将用户输入发送到语言模型 (LLM) 之前检查其中是否存在任何潜在不安全的内容。这确保只有适当的内容会被进一步处理。

  1. 接收用户输入
  2. 使用审核 API 分析输入是否存在任何问题内容。
  3. 条件处理： - 如果输入被审核检查标记，请相应地处理（例如，拒绝输入，要求用户重新措辞等）。 - 如果输入未被标记，请将其传递给 LLM 进行进一步处理。

我们将通过两个示例提示来演示此工作流程。一个用于文本，另一个用于图像。请注意，您可以在同一请求中同时传递文本和图像。

system_prompt = "You are a helpful assistant."

bad_request = "I want to hurt them. How can i do this?"
good_request = "I would kill for a cup of coffe. Where can I get one nearby?"

import asyncio

async def check_moderation_flag(expression):
    moderation_response = client.moderations.create(input=expression)
    flagged = moderation_response.results[0].flagged
    return flagged

async def get_chat_response(user_request):
    print("Getting LLM response")
    messages = [
        {"role": "system", "content": system_prompt},
        {"role": "user", "content": user_request},
    ]
    response = client.chat.completions.create(
        model=GPT_MODEL, messages=messages, temperature=0.5
    )
    print("Got LLM response")
    return response.choices[0].message.content


async def execute_chat_with_input_moderation(user_request):
    # Create tasks for moderation and chat response
    moderation_task = asyncio.create_task(check_moderation_flag(user_request))
    chat_task = asyncio.create_task(get_chat_response(user_request))

    while True:
        # Wait for either the moderation task or chat task to complete
        done, _ = await asyncio.wait(
            [moderation_task, chat_task], return_when=asyncio.FIRST_COMPLETED
        )

        # If moderation task is not completed, wait and continue to the next iteration
        if moderation_task not in done:
            await asyncio.sleep(0.1)
            continue

        # If moderation is triggered, cancel the chat task and return a message
        if moderation_task.result() == True:
            chat_task.cancel()
            print("Moderation triggered")
            return "We're sorry, but your input has been flagged as inappropriate. Please rephrase your input and try again."

        # If chat task is completed, return the chat response
        if chat_task in done:
            return chat_task.result()

        # If neither task is completed, sleep for a bit before checking again
        await asyncio.sleep(0.1)

# Call the main function with the good request - this should go through
good_response = await execute_chat_with_input_moderation(good_request)
print(good_response)

    response = client.moderations.create(

        input=[

2. 输出审核

输出审核对于控制语言模型 (LLM) 生成的内容至关重要。虽然 LLM 不应输出非法或有害内容，但设置额外的护栏以进一步确保内容保持在可接受和安全的范围内，从而增强应用程序的整体安全性和可靠性，这会很有帮助。常见的输出审核类型包括：

  • 内容质量保证： 确保生成的内容（例如文章、产品描述和教育材料）准确、信息丰富且不包含不当信息。
  • 社区标准合规性： 通过过滤仇恨言论、骚扰和其他有害内容，在在线论坛、讨论区和游戏社区中维护一个尊重和安全的环境。
  • 用户体验增强： 通过提供礼貌、相关且不包含任何不当语言或内容的响应来改善聊天机器人和自动化服务中的用户体验。

在所有这些场景中，输出审核在维护语言模型生成内容的质量和完整性方面发挥着至关重要的作用，确保其符合平台及其用户的标准和期望。

设置审核阈值

OpenAI 为审核类别选择的阈值平衡了我们用例的精度和召回率，但您的用例或对审核的容忍度可能不同。设置此阈值是优化的常见领域 - 我们建议构建一个评估集并使用混淆矩阵对结果进行评分，以设置正确的审核容忍度。这里的权衡通常是：

  • 更多的误报会导致用户体验受损，客户会感到恼火，并且助手似乎不太有用。
  • 更多的漏报可能对您的业务造成持久的损害，因为人们会让助手回答不当问题或提供不当响应。

例如，在一个专门从事创意写作的平台上，某些敏感主题的审核阈值可能会设置得更高，以允许更大的创作自由度，同时仍然提供一个安全网来捕获明显超出可接受表达范围的内容。权衡是，虽然允许了一些在其他情况下可能被认为不当的内容，但这被认为是可接受的，因为平台的用途和受众的期望。

工作流程：

我们将创建一个工作流程，其中包含审核 API，用于在将 LLM 响应显示给用户之前检查其中是否存在任何潜在不安全的内容。这确保只有适当的内容会显示给用户。

  1. 接收用户输入
  2. 将提示发送到 LLM 并生成响应
  3. 使用审核 API 分析 LLM 的响应是否存在任何问题内容。
  4. 条件处理： - 如果响应被审核检查标记，请相应地处理（例如，拒绝响应，显示占位符消息等）。 - 如果响应未被标记，请将其显示给用户。

我们将通过之前的两个示例提示来演示此工作流程。

async def execute_all_moderations(user_request):
    # Create tasks for moderation and chat response
    input_moderation_task = asyncio.create_task(check_moderation_flag(user_request))
    chat_task = asyncio.create_task(get_chat_response(user_request))

    while True:
        done, _ = await asyncio.wait(
            [input_moderation_task, chat_task], return_when=asyncio.FIRST_COMPLETED
        )

        # If input moderation is not completed, wait and continue to the next iteration
        if input_moderation_task not in done:
            await asyncio.sleep(0.1)
            continue

        # If input moderation is triggered, cancel chat task and return a message
        if input_moderation_task.result() == True:
            chat_task.cancel()
            print("Input moderation triggered")
            return "We're sorry, but your input has been flagged as inappropriate. Please rephrase your input and try again."

        # Check if chat task is completed
        if chat_task in done:
            chat_response = chat_task.result()
            output_moderation_response = await check_moderation_flag(chat_response)

            # Check if output moderation is triggered
            if output_moderation_response == True:
                print("Moderation flagged for LLM response.")
                return "Sorry, we're not permitted to give this answer. I can help you with any general queries you might have."

            print('Passed moderation')
            return chat_response

        # If neither task is completed, sleep for a bit before checking again
        await asyncio.sleep(0.1)

3. 自定义审核

注意：本节不使用审核 API。

自定义审核提供了一种定制的内容过滤方法，允许精确遵守特定的社区标准或主题，使其成为利基平台或专业内容的理想选择。与审核 API 涵盖的通用类别不同，自定义审核提供了更高的控制级别，并且需要更多的设置和维护工作。它与护栏共享确保内容安全这一共同目标，专注于设置用于过滤和管理内容的标准，无论是用户输入还是语言模型的输出。自定义审核可以作为审核 API 的替代或补充，它使用我们的 completions 端点。对于有兴趣进一步探索护栏的人来说，可以在此处找到包含示例和指南的食谱 here。

工作流程

我们将实现一个小型示例来展示自定义审核。我们将使用额外的 gpt-4o mini 调用来评估消息并决定是否应审核它们。

def custom_moderation(content, parameters):
    # Define the prompt 
    prompt = f"""Please assess the following content for any inappropriate material. You should base your assessment on the given parameters.
    Your answer should be in json format with the following fields: 

        - flagged: a boolean indicating whether the content is flagged for any of the categories in the parameters
        - reason: a string explaining the reason for the flag, if any
        - parameters: a dictionary of the parameters used for the assessment and their values
    Parameters: {parameters}\n\nContent:\n{content}\n\nAssessment:"""

    # Call model with the prompt
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        response_format={ "type": "json_object" },
        messages=[
            {"role": "system", "content": "You are a content moderation assistant."},
            {"role": "user", "content": prompt}
        ]
    )

    # Extract the assessment from the response
    assessment = response.choices[0].message.content

    return assessment
