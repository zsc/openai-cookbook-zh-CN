优化检索增强生成（RAG）中的GPT-4o视觉模态应用

在处理包含大量图像、图表和表格的文档时，实现检索增强生成（RAG）会带来独特的挑战。传统的RAG模型在处理文本数据方面表现出色，但在视觉元素起关键信息传递作用时，往往会力不从心。在本指南中，我们通过利用视觉模态来提取和解释视觉内容，弥合了这一差距，确保生成的响应尽可能信息丰富和准确。

我们的方法包括将文档解析为图像，并利用元数据标记来识别包含图像、图表和表格的页面。当语义搜索检索到此类页面时，我们会将页面图像传递给视觉模型，而不是仅仅依赖文本。这种方法增强了模型理解和回答与视觉数据相关的用户查询的能力。

在本指南中，我们将探讨并演示以下关键概念：

1. 使用Pinecone设置向量存储：
  • 了解如何初始化和配置Pinecone以高效存储向量嵌入。
2. 解析PDF并提取视觉信息：
  • 探索将PDF页面转换为图像的技术。
  • 使用GPT-4o视觉模态从包含图像、图表或表格的页面中提取文本信息。
3. 生成嵌入：
  • 利用嵌入模型为文本数据创建向量表示。
  • 标记包含视觉内容的页面，以便我们在向量存储上设置元数据标志，并检索图像以通过视觉模态传递给GPT-4o。
4. 将嵌入上传到Pinecone：
  • 将这些嵌入上传到Pinecone进行存储和检索。
5. 执行相关页面语义搜索：
  • 对页面文本执行语义搜索，以查找与用户查询最匹配的页面。
  • 将匹配的页面文本提供给GPT-4o作为上下文来回答用户的问题。
6. 处理包含视觉内容的页面（可选步骤）：
  • 了解如何使用GPT-4o视觉模态传递图像以进行问答并提供额外上下文。
  • 理解此过程如何提高涉及视觉数据的响应准确性。

通过本指南的学习，您将对如何实现能够处理和解释包含复杂视觉元素的文档的RAG系统有深入的了解。这些知识将使您能够构建提供更丰富、更准确信息的AI解决方案，从而提高用户满意度和参与度。

我们将使用世界银行的报告——“建设一个更美好的银行，创造一个更美好的世界：2024年年度报告” 来阐述这些概念，因为该文档包含图像、表格和图表数据的混合。

请记住，使用视觉模态资源消耗大，会导致延迟和成本增加。建议仅在纯文本提取方法的评估基准性能不佳的情况下使用视觉模态。有了这个背景，让我们开始吧。

第1步：使用Pinecone设置向量存储

在本节中，我们将使用Pinecone设置一个向量存储，以高效地存储和管理我们的嵌入。Pinecone是一个为处理高维向量数据而优化的向量数据库，这对于语义搜索和相似性匹配等任务至关重要。

先决条件

  1. 注册Pinecone并按照此处的说明获取API密钥Pinecone数据库快速入门
  2. 使用pip install "pinecone[grpc]"安装Pinecone SDK。gRPC（gRPC远程过程调用）是一个高性能、开源的通用RPC框架，它使用HTTP/2进行传输，使用Protocol Buffers（protobuf）作为接口定义语言，并支持分布式系统中的客户端-服务器通信。它旨在使服务间通信更高效，并适用于微服务架构。

安全地存储API密钥

  1. 出于安全目的，请将API密钥存储在项目目录中的.env文件中，如下所示： PINECONE_API_KEY=your-api-key-here。

  2. 安装pip install python-dotenv以从.env文件读取API密钥。

创建Pinecone索引 我们将使用create_index函数在Pinecone上初始化我们的嵌入数据库。有两个关键参数需要考虑：

  1. 维度：这必须与您选择的模型生成的嵌入的维度相匹配。例如，OpenAI的text-embedding-ada-002模型生成的嵌入维度为1536，而text-embedding-3-large模型生成的嵌入维度为3072。我们将使用text-embedding-3-large模型，因此我们将维度设置为3072。

  2. 度量：距离度量决定了向量之间的相似性如何计算。Pinecone支持多种度量，包括余弦、点积和欧几里得。在本指南中，我们将使用余弦相似度度量。您可以在Pinecone距离度量文档中了解更多关于距离度量的信息。

import os
import time
# 导入Pinecone库
from pinecone.grpc import PineconeGRPC as Pinecone
from pinecone import ServerlessSpec

from dotenv import load_dotenv

load_dotenv()

api_key = os.getenv("PINECONE_API_KEY")

# 使用您的API密钥初始化Pinecone客户端
pc = Pinecone(api_key)

# 创建一个无服务器索引
index_name = "my-test-index"

if not pc.has_index(index_name):
    pc.create_index(
        name=index_name,
        dimension=3072,
        metric="cosine",
        spec=ServerlessSpec(
            cloud='aws',
            region='us-east-1'
        )
    )

# 等待索引准备就绪
while not pc.describe_index(index_name).status['ready']:
    time.sleep(1)

导航到Pinecone上的索引列表，您应该能够看到索引列表中有my-test-index。

第2步：解析PDF并提取视觉信息：

在本节中，我们将解析PDF文档——世界银行报告——“建设一个更美好的银行，创造一个更美好的世界：2024年年度报告”，并提取文本和视觉信息，例如描述图像、图表和表格。该过程包括三个主要步骤：

  1. 将PDF解析为单个页面：我们将PDF拆分为单独的页面，以便于处理。
  2. 将PDF页面转换为图像：这使得视觉GPT-4o视觉能力能够将页面作为图像进行分析。
  3. 处理图像和表格：向GPT-4o提供指令以提取文本，并描述文档中的图像、图表或表格。

先决条件

在继续之前，请确保您已安装以下软件包。还要确保您的OpenAI API密钥已设置为环境变量。您可能还需要安装Poppler用于PDF渲染。

from pdf2image import convert_from_bytes
from io import BytesIO

# OpenAI客户端
oai_client = OpenAI()


# 将文档分块为单页块
def chunk_document(document_url):
    # 下载PDF文档
    response = requests.get(document_url)
    pdf_data = response.content

    # 使用PyPDF2读取PDF数据
    pdf_reader = PdfReader(BytesIO(pdf_data))
    page_chunks = []

    for page_number, page in enumerate(pdf_reader.pages, start=1):
        pdf_writer = PdfWriter()
        pdf_writer.add_page(page)
        pdf_bytes_io = BytesIO()
        pdf_writer.write(pdf_bytes_io)
        pdf_bytes_io.seek(0)
        pdf_bytes = pdf_bytes_io.read()
        page_chunk = {
            'pageNumber': page_number,
            'pdfBytes': pdf_bytes
        }
        page_chunks.append(page_chunk)

    return page_chunks


# 编码图像的函数
def encode_image(local_image_path):
    with open(local_image_path, "rb") as image_file:
        return base64.b64encode(image_file.read()).decode('utf-8')


# 将页面转换为图像的函数
def convert_page_to_image(pdf_bytes, page_number):
    # 将PDF页面转换为图像
    images = convert_from_bytes(pdf_bytes)
    image = images[0]  # 应该只有一页

    # 定义保存图像的目录（相对于您的脚本）
    images_dir = 'images'  # 在此处使用相对路径

    # 确保目录存在
    os.makedirs(images_dir, exist_ok=True)

    # 将图像保存到图像目录
    image_file_name = f"page_{page_number}.png"
    image_file_path = os.path.join(images_dir, image_file_name)
    image.save(image_file_path, 'PNG')

    # 返回相对图像路径
    return image_file_path


# 将图像传递给LLM进行解释
def get_vision_response(prompt, image_path):
    # 获取base64字符串
    base64_image = encode_image(image_path)

    response = oai_client.chat.completions.create(
        model="gpt-4o",
        messages=[
            {
                "role": "user",
                "content": [
                    {"type": "text", "text": prompt},
                    {
                        "type": "image_url",
                        "image_url": {
                            "url": f"data:image/jpeg;base64,{base64_image}"
                        },
                    },
                ],
            }
        ],
    )
    return response


# 整合所有内容的文档处理函数
def process_document(document_url):
    try:
        # 更新文档状态为“处理中”
        print("Document processing started")

        # 获取每页的块
        page_chunks = chunk_document(document_url)
        total_pages = len(page_chunks)

        # 准备一个列表来收集页面数据
        page_data_list = []

        # 在此处添加进度条
        for page_chunk in tqdm(page_chunks, total=total_pages, desc='Processing Pages'):
            page_number = page_chunk['pageNumber']
            pdf_bytes = page_chunk['pdfBytes']

            # 将页面转换为图像
            image_path = convert_page_to_image(pdf_bytes, page_number)

            # 为视觉API准备问题
            system_prompt = (
                "用户将提供您一份文档文件的图像。请执行以下操作： "
                "1. 转录页面上的文本。**文本转录：**"
                "2. 如果有图表，请描述图像并包含文本 **图像或图表描述**"
                "3. 如果有表格，请转录表格并包含文本 **表格转录**"
            )

            # 获取视觉API响应
            vision_response = get_vision_response(system_prompt, image_path)

            # 从视觉响应中提取文本
            text = vision_response.choices[0].message.content

            # 收集页面数据
            page_data = {
                'PageNumber': page_number,
                'ImagePath': image_path,
                'PageText': text
            }
            page_data_list.append(page_data)

        # 从页面数据创建DataFrame
        pdf_df = pd.DataFrame(page_data_list)
        print("Document processing completed.")
        print("DataFrame created with page data.")

        # 返回DataFrame
        return pdf_df

    except Exception as err:
        print(f"Error processing document: {err}")

步骤分解：

第5步：执行相关页面语义搜索：

在本节中，我们将实现一个语义搜索来查找文档中最相关的页面，以回答用户的问题。这种方法使用存储在Pinecone向量数据库中的嵌入，根据页面内容与用户查询的语义相似性来检索页面。通过这样做，我们可以有效地搜索文本内容，并将其作为上下文提供给GPT-4o来回答用户的问题。

步骤分解：

  1. 为用户问题生成嵌入
  • 我们使用OpenAI的嵌入模型为用户的问题生成高维向量表示。
  • 该向量捕获了问题的语义含义，使我们能够与存储的嵌入进行高效的相似性搜索。
  • 嵌入对于确保搜索查询在语义上与文档内容对齐至关重要，即使确切的词语不匹配。
  1. 查询Pinecone索引以获取相关页面
  • 使用生成的嵌入，我们查询Pinecone索引以查找最相关的页面。
  • Pinecone通过使用cosine相似性将问题嵌入与向量数据库中存储的嵌入进行比较来执行相似性搜索。如果您还记得，我们在第1步创建Pinecone数据库时已将其设置为metric参数。
  • 我们指定要检索的顶级匹配数量，通常基于覆盖范围和相关性之间的平衡。例如，检索前3-5页通常足以提供全面的答案，而不会让模型因上下文过多而负担过重。
  1. 编译匹配页面的元数据以提供上下文
  • 一旦确定了相关的嵌入，我们就收集其关联的元数据，包括提取的文本和页码。
  • 这些元数据对于构建提供给GPT-4o的上下文至关重要。
  • 我们还将编译的信息格式化为JSON，以便LLM轻松解释。
  1. 使用GPT-4o模型生成答案
  • 最后，我们将编译的上下文传递给GPT-4o。
  • 模型使用上下文生成信息丰富、连贯且与上下文相关的答案。
  • 检索到的上下文有助于LLM更准确地回答问题，因为它能够访问文档中的相关信息。

# 获取用户问题响应的函数
def get_response_to_question(user_question, pc_index):
    # 获取问题的嵌入，以查找包含信息的相关页面
    question_embedding = get_embedding(user_question)

    # 获取响应向量嵌入
    response = pc_index.query(
        vector=question_embedding,
        top_k=2,
        include_values=True,
        include_metadata=True
    )

    # 从匹配项中收集元数据
    context_metadata = [match['metadata'] for match in response['matches']]

    # 将元数据字典列表转换为提示的JSON字符串
    context_json = json.dumps(context_metadata, indent=3)

    prompt = f"""您是一位乐于助人的助手。请使用以下上下文和图像回答问题。在答案中，请在<source></source>标签中包含您找到信息的文档参考和页码。如果您找不到信息，可以说“我找不到信息”。

    问题：{user_question}

    <SOURCES>
    {context_json}
    </SOURCES>
    """

    # 使用提示调用completions端点
    completion = oai_client.chat.completions.create(
        model="gpt-4o",
        messages=[
            {"role": "system", "content": prompt}
        ]
    )

    return completion.choices[0].message.content

当元数据指示存在图像、图表或表格时，我们可以将图像作为上下文传递给GPT-4o，而不是提取的文本。当视觉信息的文本描述不足以传达上下文时，这种方法可能很有用。例如，复杂的图形，如工程图或复杂的图表，可能就是这种情况。
