▶ one_way_translation_using_realtime_api
📄 Translation Demo

在本指南中，您将学习如何使用 OpenAI Embedding API 来生成语言嵌入，然后将这些嵌入索引到 Pinecone 向量数据库中，以实现快速可扩展的向量搜索。

这是构建语义搜索、问答、威胁检测以及其他依赖于自然语言处理和大规模文本语料库搜索的应用程序的强大且常见的组合。

基本工作流程如下：

嵌入和索引

  • 使用 OpenAI Embedding API 为您的文档（或任何文本数据）生成向量嵌入。
  • 将这些向量嵌入上传到 Pinecone，它可以存储和索引数百万/数十亿个向量嵌入，并以超低延迟进行搜索。

搜索

  • 再次通过 OpenAI Embedding API 处理您的查询文本或文档。
  • 获取生成的向量嵌入，并将其作为查询发送到 Pinecone。
  • 获取语义上相似的文档，即使它们与查询没有共享任何关键字。

然后我们使用 OpenAI 为每个短语创建向量嵌入，并将每个短语的 ID、向量嵌入和原始文本 upsert 到 Pinecone。

from tqdm.auto import tqdm

count = 0  # 我们将使用计数来创建唯一的 ID
batch_size = 32  # 以 32 为一批处理所有内容
for i in tqdm(range(0, len(trec['text']), batch_size)):
    # 设置批次的结束位置
    i_end = min(i+batch_size, len(trec['text']))
    # 获取批次的行和 ID
    lines_batch = trec['text'][i: i+batch_size]
    ids_batch = [str(n) for n in range(i, i_end)]
    # 创建嵌入
    res = client.embeddings.create(input=lines_batch, model=MODEL)
    embeds = [record.embedding for record in res.data]
    # 准备元数据并 upsert 批次
    meta = [{'text': line} for line in lines_batch]
    to_upsert = zip(ids_batch, embeds, meta)
    # upsert 到 Pinecone
    index.upsert(vectors=list(to_upsert))
