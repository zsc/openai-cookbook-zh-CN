多智能体编排与OpenAI Agents SDK：金融投资组合分析示例

引言

本指南面向已熟悉OpenAI模型和LLM智能体，并希望了解如何为真实、复杂的任务编排智能体团队的读者。

您将学到什么

在本Notebook中，您将学习如何使用OpenAI Agents SDK设计和实现一个复杂的、多智能体协作系统。具体来说，您将了解如何：

  • 构建一个工作流，其中多个专业智能体（宏观、基本面、量化）在一个投资组合经理智能体的协调下进行协作，以解决具有挑战性的投资研究问题。
  • 使用“智能体即工具”的方法，其中一个中心智能体协调并像调用工具一样调用其他智能体来执行特定的子任务。
  • 在一个单一的、集成的流程中利用SDK支持的所有主要工具类型（自定义Python函数、代码解释器和Web搜索等托管工具，以及外部MCP服务器）。
  • 应用智能体模式中的模块化、并行化和可观察性的最佳实践。

为什么这很重要

“智能体即工具”模式是构建透明、可审计和可扩展的多智能体协作的强大方法。本示例演示了如何使用OpenAI Agents SDK结合深度专业化、并行执行和强大的编排。

在本指南结束时，您将拥有一个清晰的蓝图，用于构建自己的多智能体工作流，以进行研究、分析或任何受益于专家协作的复杂任务。

什么是多智能体协作？

多智能体协作是指多个自主智能体（LLM“节点”）协调以实现单个智能体难以处理的总体目标。每个智能体处理特定的子任务或专业领域，而不是一个单一的整体提示，一个编排层将这些智能体“节点”连接成一个连贯的工作流。这种方法对于复杂系统很有用——例如，金融分析可以分解为宏观经济分析、公司基本面分析和量化信号分析，每个由不同的智能体专家处理。智能体共享信息，并将它们的结果结合起来以产生最终结果。

协作模式：交接 vs. 智能体即工具

OpenAI Agents SDK支持多种智能体协同工作模式：

  • 交接协作： 一个智能体可以在问题中间 交接 控制权给另一个智能体。在交接架构中，每个智能体都知道其他智能体，并可以决定何时将任务委托给更合适的智能体。这对于开放式或对话式工作流非常灵活，但可能难以维护任务的全局视图。在SDK文档中了解更多。

  • 智能体即工具： 在这种方法中，一个智能体（通常是中央规划师或管理器）像调用工具一样调用其他智能体。子智能体不接管对话；相反，主智能体为特定子任务调用它们并将它们的结果整合进来。这种模型保持了单一的控制线程（主智能体协调一切），并且倾向于简化协调。本代码库使用“智能体即工具”模型：投资组合经理智能体保持主导地位，在需要专业知识时将其他专业智能体用作工具。这个选择保持了整体推理的透明度，并允许子任务的并行执行，这对于复杂的分析非常理想。

有关这些协作模式的更多信息，请参阅OpenAI Agents SDK文档。

架构概述

我们的系统遵循中心辐射设计。投资组合经理智能体是中心（中央协调器），而专业智能体是辐射。用户的查询（例如，“计划的降息将如何影响我的GOOGL持仓？”）首先发送给投资组合经理。投资组合经理智能体被提示分解问题并委托给适当的专业智能体。它将每个专业智能体视为一个可调用的工具，在需要它们的专业知识时调用它们。所有三个都向投资组合经理报告，然后投资组合经理综合最终答案给用户。

支持的工具类型

Agents SDK的一个关键优势在于定义智能体可使用的工具的灵活性。工具的范围可以从简单的Python函数到外部服务。在此项目中，我们使用：

  • MCP（模型上下文协议）服务器： 用于以标准化的方式将智能体连接到外部工具和数据源。本项目使用本地MCP服务器来获取雅虎财经数据（参见mcp/yahoo_finance_server.py）。了解更多：OpenAI MCP文档 | MCP规范

  • OpenAI托管工具： 托管工具是OpenAI提供的内置工具，无需自定义实现。它们开箱即用地提供强大的功能，例如代码解释器（用于量化/统计分析）和Web搜索（用于最新新闻和数据）。这些工具易于集成，由OpenAI维护，并允许智能体执行代码执行和实时信息检索等高级操作，而无需额外设置。

  • 自定义工具： 自定义工具是您定义的任何Python函数，并将其注册为智能体的工具。Agents SDK使此操作变得容易：只需装饰您的函数，SDK就会自动提取其名称、文档字符串和输入模式。这对于领域特定逻辑、数据访问或工作流扩展非常理想。 在我们的项目中，我们使用自定义工具来访问FRED经济数据（参见FRED API）并执行文件系统操作。

自定义工具为您提供了在内置或托管工具之外扩展智能体能力的完全灵活性。参见SDK关于函数工具的文档。

想添加更多工具？ SDK支持广泛的工具类型，包括Web搜索、文件搜索、代码执行等。请参阅SDK文档中支持的工具完整列表。

分解首席投资组合经理智能体

首席投资组合经理（PM）智能体是整个工作流的编排者。它协调一组四个专业智能体，每个智能体专注于不同的专业领域。这种设计是故意的：让一个单一的智能体承担所有可能的职责会导致肤浅、通用的输出，并且难以维护或改进您的系统。

为什么采用这种设计？

通过将问题分解为专业智能体——每个智能体都有明确的角色——您可以获得：

  • 更深入、更高质量的研究： 每个智能体都可以专注于其领域，使用正确的工具和提示来完成工作。PM智能体将这些观点汇集起来，以获得更细致、更强大的答案。

  • 模块化和清晰性： 您可以更新、测试或改进一个智能体而不影响其他智能体。这使得您的系统在不断发展的需求下更易于维护和扩展。

  • 通过并行化实现更快的速度： 独立的智能体可以同时工作，大大缩短完成复杂的多部分分析所需的时间。

  • 一致性和可审计性： 一个结构化的、由提示驱动的工作流确保每次运行都遵循最佳实践，易于调试，并产生您可以信任和审查的输出。

这种方法非常适合任何您想要深度、专业化和可靠性的应用程序——无论您是构建研究助手、决策支持工具，还是任何受益于专家协作和编排的系统。

我们在实践中如何实现这一点：

  • 每个专业智能体（基本面、宏观、量化）都使用SDK的function_tool装饰器包装成一个可调用的工具，并带有自定义名称和描述。这使得PM智能体的工具集明确且对LLM友好。

  • 首席PM智能体使用run_all_specialists_parallel工具并发调用所有三个专业智能体，利用parallel_tool_calls=True以实现最大速度和效率。

  • 智能体的提示从Markdown文件（pm_base.md）加载，该文件不仅编码了公司的理念，还编码了详细的工具使用规则和分步工作流。这确保了每次运行都是一致的、可审计的，并符合最佳实践。

  • 在收集和审查专业输出后，PM智能体使用专用的备忘录编辑工具来组装、格式化和最终确定投资报告。这种关注点分离使工作流保持模块化且易于扩展。

  • 该系统设计为可扩展的：您可以添加新的专业智能体，替换工具，或更新提示而不破坏整体编排逻辑。所有工具调用、智能体决策和输出都捕获在OpenAI Traces中，以实现完全的透明度和调试。

这些实现选择直接支持上述好处——实现深度、模块化和可靠的多智能体研究工作流，这些工作流易于维护、审计和改进。
