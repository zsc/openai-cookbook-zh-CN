视觉微调 GPT-4o 以实现视觉问答

我们很高兴地宣布推出 视觉微调 GPT-4o，这是一种尖端的跨媒体微调功能，使开发人员能够使用图像和文本对 GPT-4o 进行微调。借助此新功能，您可以自定义模型以获得更强的图像理解能力，从而为各行各业和各种应用解锁可能性。

从高级视觉搜索到自动驾驶汽车或智能城市的改进对象检测，视觉微调使您能够根据特定需求量身定制解决方案。通过结合文本和图像输入，该产品非常适合视觉问答等任务，这些任务通过分析图像得出详细的、上下文感知的答案。总的来说，当模型接收到的问题和图像与训练数据相似时，这似乎是最有效的，因为我们可以教会模型如何搜索和识别图像的相关部分以正确回答问题。与仅基于文本输入的微调类似，视觉微调对于教授模型新信息并不那么有用。

在本指南中，我们将引导您完成使用多模态输入对 GPT-4o 进行微调的步骤。具体来说，我们将演示如何训练模型来回答与书籍图像相关的问题，但潜在的应用涵盖无数领域——从网页设计和教育到医疗保健和研究。

无论您是希望为制造业构建更智能的缺陷检测模型，增强复杂的文档处理和图表理解能力，还是为各种其他用例开发具有更好视觉理解能力的应用，本指南都将向您展示入门是多么快速和容易。

有关更多信息，请查看完整的文档。

from openai import OpenAI, ChatCompletion

加载数据集

我们将使用来自 OCR-VQA 数据集 的书籍图像上的问答对数据集，该数据集可通过 HuggingFace 访问。此数据集包含 207,572 张书籍图像以及相关的问答对，询问书籍的标题、作者、版本和年份以及类型。总而言之，该数据集包含约 100 万个问答对。在本指南中，我们将仅使用该数据集的一小部分来训练、验证和测试我们的模型。

我们相信此数据集非常适合在多模态输入上进行微调，因为它不仅要求模型准确识别相关边界框以提取关键信息，还需要推理图像内容以正确回答问题。

from datasets import load_dataset

# 加载数据集
ds = load_dataset("howard-hou/OCR-VQA")

我们将首先抽取 150 个训练示例、50 个验证示例和 100 个测试示例。我们还将展开 questions 和 answers 列，为每一行创建一个单独的问答对。此外，由于我们的图像存储为字节字符串，我们将把它们转换为图像进行处理。

import pandas as pd
from io import BytesIO
from PIL import Image

# 抽取 150 个训练示例、50 个验证示例和 100 个测试示例
ds_train = ds['train'].shuffle(seed=42).select(range(150))
ds_val = ds['validation'].shuffle(seed=42).select(range(50))
ds_test = ds['test'].shuffle(seed=42).select(range(100))

# 转换为 pandas 数据框
ds_train = ds_train.to_pandas()
ds_val = ds_val.to_pandas()
ds_test = ds_test.to_pandas()

# 将字节字符串转换为图像
ds_train['image'] = ds_train['image'].apply(lambda x: Image.open(BytesIO(x['bytes'])))
ds_val['image'] = ds_val['image'].apply(lambda x: Image.open(BytesIO(x['bytes'])))
ds_test['image'] = ds_test['image'].apply(lambda x: Image.open(BytesIO(x['bytes'])))

# 展开 'questions' 和 'answers' 列
ds_train = ds_train.explode(['questions', 'answers'])
ds_val = ds_val.explode(['questions', 'answers'])
ds_test = ds_test.explode(['questions', 'answers'])

# 重命名列
ds_train = ds_train.rename(columns={'questions': 'question', 'answers': 'answer'})
ds_val = ds_val.rename(columns={'questions': 'question', 'answers': 'answer'})
ds_test = ds_test.rename(columns={'questions': 'question', 'answers': 'answer'})

# 为每个示例创建唯一 ID
ds_train = ds_train.reset_index(drop=True)
ds_val = ds_val.reset_index(drop=True)
ds_test = ds_test.reset_index(drop=True)

# 选择列
ds_train = ds_train[['question', 'answer', 'image']]
ds_val = ds_val[['question', 'answer', 'image']]
ds_test = ds_test[['question', 'answer', 'image']]

让我们检查一下训练集中的一个随机样本。

在此示例中，问题提示模型确定书籍的标题。在这种情况下，答案相当模糊，因为存在主要标题“Patty's Patterns - Advanced Series Vol. 1 & 2”以及副标题“100 Full-Page Patterns Value Bundle”，它们位于图像的不同部分。此外，这里的作者姓名不是个人，而是一个名为“Penny Farthing Graphics”的团体，这可能会被误认为是标题的一部分。

这种类型的任务在视觉问答中很常见，模型必须解释复杂的图像并提供准确的、特定于上下文的响应。通过训练这些类型的问题，我们可以增强模型在各种领域进行详细图像分析的能力。

数据准备

为确保成功微调我们的模型，正确构建训练数据至关重要。正确格式化数据有助于避免训练期间的验证错误，并确保模型能够有效地从文本和图像输入中学习。好消息是，这个过程相当简单。

训练数据集中的每个示例都应采用与聊天补全 API 相同的格式的对话。具体来说，这意味着将数据构建为一系列消息，其中每条消息都包含一个 role（例如，“user”或“assistant”）和消息的 content。

由于我们同时处理文本和图像进行视觉微调，因此我们将构建这些消息以同时包含这两种内容类型。对于每个训练样本，关于图像的问题都作为用户消息呈现，相应的答案作为助手消息提供。

图像可以通过以下两种方式之一包含：

  • 作为HTTP URL，引用图像的位置。
  • 作为包含以base64编码的图像的数据 URL。

以下是消息格式应如何显示：

{
    "messages": 
    [
        {
            "role": "system",
            "content": "Use the image to answer the question."
        },

为确保我们的图像正确格式化以进行视觉微调，它们必须采用base64 格式，并且可以是RGB 或 RGBA。这可确保模型在训练期间能够准确处理图像。下面是一个处理图像编码的函数，同时在必要时将其转换为正确的格式。

此函数允许我们控制图像编码的质量，如果我们想减小文件大小，这会很有用。100 是最高质量，1 是最低质量。微调作业的最大文件大小为 1GB，但我们不太可能通过大量训练数据看到改进。尽管如此，我们可以使用 quality 参数来减小文件大小，以适应文件大小限制。

import base64

def encode_image(image, quality=100):
    if image.mode != 'RGB':
        image = image.convert('RGB')  # 转换为 RGB
    buffered = BytesIO()
    image.save(buffered, format="JPEG", quality=quality) 
    return base64.b64encode(buffered.getvalue()).decode("utf-8")

为了评估我们的模型响应，我们将使用 GPT-4o 来确定预测响应与真实响应之间的相似性。我们将根据以下标准对预测答案进行排名：

  • 非常相似：预测答案与真实答案完全匹配，并且没有遗漏重要信息，尽管可能存在一些细微的遗漏或标点符号上的差异。

  • 大部分相似：预测答案与真实答案非常吻合，可能缺少一些单词或短语。

  • 有些相似：尽管预测答案与真实答案有明显的差异，但核心内容是准确且语义相似的，可能缺少一些信息。

  • 不正确：预测答案完全不正确、不相关，或包含关键错误或遗漏真实答案的信息。

  • 更好的泛化能力：虽然未经微调的 GPT-4o 能够对许多问题至少在某种程度上给出接近真实答案的回答，但它的一致性较差。由于在训练期间接触了多模态数据，微调模型在各种测试问题上表现出更好的泛化能力。

尽管视觉微调的结果很有希望，但仍有改进的空间。与文本微调类似，视觉微调的有效性在很大程度上取决于训练数据的质量、多样性和代表性。特别是，模型受益于专注于最常出现错误的案例，从而可以进行有针对性的改进。

在审查不正确的结果时，微调模型的大多数“不正确”响应实际上是由于数据集中标签不一致造成的。例如，一些真实答案仅提供作者的名字和姓氏，而图像实际上也显示了中间名首字母。同样，一些标题的真实标签包含副标题和标语，而另一些则不包含。

另一个常见的主题是类型分类错误。尽管模型几乎总能为真实答案提供语义上相似的类型，但有时答案会偏离。这可能是由于训练数据中缺少这些类型。为模型提供更多样化的训练示例来涵盖这些类型，或为处理边缘情况提供更清晰的说明，可以帮助指导模型的理解。

后续步骤：

  • 扩展训练数据集：添加更多涵盖模型较弱领域的各种示例，例如识别类型，可以显著提高性能。

  • 专家指导的提示：将特定领域的说明纳入训练提示可以进一步优化模型在复杂情况下准确解释和响应的能力。
